# 第6章：静态网格体与模型

> 🎯 **本章目标**：理解静态网格体概念，掌握模型导入、碰撞、LOD设置

---

## 一、什么是静态网格体？

### 1.1 通俗解释

**静态网格体（Static Mesh）** = 不会动的3D模型

```
类比理解：

【静态网格体】                    【骨骼网格体】
┌─────────────┐                  ┌─────────────┐
│ • 房子      │                  │ • 人物      │
│ • 石头      │                  │ • 动物      │
│ • 树木      │                  │ • 会开门    │
│ • 椅子      │                  │             │
├─────────────┤                  ├─────────────┤
│ 不会变形    │                  │ 有骨骼      │
│ 不会动      │                  │ 可以动画    │
│ 性能好      │                  │ 性能开销大  │
└─────────────┘                  └─────────────┘

【简单记忆】
不会动 = Static Mesh
会动/会变形 = Skeletal Mesh
```

### 1.2 静态网格体的组成

```
┌─────────────────────────────────────────────┐
│              静态网格体构成                   │
├─────────────────────────────────────────────┤
│                                             │
│    ┌─────────────────────────────────┐     │
│    │                                 │     │
│    │      📦 3D几何体 (顶点+面)      │     │
│    │         - 顶点位置               │     │
│    │         - 面片（三角形）         │     │
│    │         - UV坐标                 │     │
│    │                                 │     │
│    │      🎨 材质插槽                │     │
│    │         - 可以赋予材质           │     │
│    │         - 一个模型可有多个       │     │
│    │                                 │     │
│    │      💥 碰撞体 (Collision)      │     │
│    │         - 物理计算用             │     │
│    │         - 看不到但摸得到         │     │
│    │                                 │     │
│    │      📐 LOD (多细节层次)        │     │
│    │         - 远处简化模型           │     │
│    │         - 节省性能               │     │
│    │                                 │     │
│    └─────────────────────────────────┘     │
│                                             │
└─────────────────────────────────────────────┘
```

## 二、模型导入全流程

### 2.1 准备模型

**建模软件导出设置（Blender为例）**：

| 设置项 | 推荐值 | 说明 |
|--------|--------|------|
| 缩放 | 全部设为1 | 避免大小错乱 |
| 应用变换 | 勾选 | 应用缩放和旋转 |
| 前进轴 | -Z | UE坐标系 |
| 向上轴 | Y | UE坐标系 |
| 三角形化 | 勾选 | 确保是三角形 |

**命名规范**：
```
✅ SM_Chair_01.fbx
✅ SM_Tree_Pine.fbx
❌ chair.fbx（太简单）
❌ 我的椅子.fbx（中文可能出错）
```

### 2.2 导入步骤详解

```
步骤1：拖入FBX文件到内容浏览器

步骤2：配置导入选项

┌─────────────────────────────────────────────────┐
│ FBX 导入选项                                     │
├─────────────────────────────────────────────────┤
│ 【常规】                                         │
│   静态网格体：☑                                  │
│   骨骼网格体：☐ （只有角色才勾）                │
│   动画：☐                                       │
│                                                 │
│ 【网格体】                                       │
│   ☑ 生成碰撞                                    │
│   ☑ 转换场景单位                                │
│   ☑ 烘焙轴定向到顶点                            │
│                                                 │
│ 【材质】                                         │
│   ☑ 导入材质和纹理                              │
│   ☑ 反转普通贴图导入                            │
│                                                 │
│ 【变换】                                         │
│   统一缩放：1.0                                 │
└─────────────────────────────────────────────────┘

步骤3：点击"导入"或"导入所有"

步骤4：检查导入结果
  ├── 双击打开静态网格体编辑器
  ├── 检查模型是否正常
  └── 检查碰撞是否生成
```

### 2.3 导入后检查清单

```
✅ 检查清单：

1. 大小是否正常？
   └── 拖入场景，和默认地板对比

2. 材质是否正确？
   └── 双击静态网格体 → 检查材质槽
   └── 紫色 = 丢失材质，需要重新指定

3. 碰撞是否生成？
   └── 视口 → 显示 → 碰撞
   └── 应该看到绿色线框包裹模型

4. 法线是否正确？
   └── 看模型是否有"黑斑"
   └── 有 = 法线翻转了，需在建模软件修复

5. UV是否正常？
   └── 静态网格体编辑器 → UV
   └── 检查UV是否重叠、是否在0-1空间
```

## 四、碰撞设置

### 4.1 为什么需要碰撞？

```
没有碰撞的情况：                    有碰撞的情况：
┌─────────────────┐                ┌─────────────────┐
│                 │                │                 │
│   玩家角色      │                │   玩家角色      │
│      ↓          │                │      ↓          │
│   ═══════════   │                │   ┌─────────┐   │
│      墙壁       │                │   │ 碰撞体  │   │
│                 │                │   └─────────┘   │
│  直接穿过！     │                │      墙壁       │
└─────────────────┘                │  被挡住 ✓       │
                                   └─────────────────┘
```

### 4.2 碰撞生成方式

| 生成方式 | 用途 | 推荐度 |
|----------|------|--------|
| **自动凸包碰撞** | 自动生成包围盒 | ⭐⭐⭐⭐⭐ |
| **简化凸包** | 减少复杂度 | ⭐⭐⭐⭐ |
| **添加盒体简化** | 长方体盒子 | ⭐⭐⭐⭐ |
| **添加球体简化** | 球形 | ⭐⭐⭐ |
| **添加胶囊体** | 胶囊形状 | ⭐⭐⭐ |
| **复杂碰撞** | 精确网格 | ⭐⭐ |

### 4.3 碰撞复杂度设置

```
细节面板 → Collision → Collision Complexity

【CTF_UseDefault】
   使用项目默认设置

【CTF_UseSimpleAndComplex】 ⭐最常用
   简单碰撞：用于物理计算（如角色移动）
   复杂碰撞：用于命中检测（如子弹射击）

【CTF_UseSimpleAsComplex】
   只使用简单碰撞，性能更好

【CTF_UseComplexAsSimple】
   使用复杂碰撞作为简单碰撞
   最精确但性能开销大
```

## 五、LOD（多细节层次）

### 5.1 什么是LOD？

```
LOD = Level of Detail = 细节层次

原理：
┌──────────────────────────────────────────────────────┐
│                                                      │
│  玩家位置                                            │
│      ↓                                               │
│                                                      │
│   【近距离】              【远距离】                  │
│   ┌───────┐              ┌───┐                      │
│   │ ┌─┐   │              │ □ │  ← 简化的模型        │
│   │ │ │   │  5000面       └───┘  100面              │
│   │ └─┘   │                                        │
│   └───────┘                                        │
│                                                      │
│   显示高面数模型          显示低面数模型             │
│   细节丰富                节省性能                   │
│                                                      │
└──────────────────────────────────────────────────────┘

好处：
• 远处看不到细节，用简单模型
• 近处需要细节，用复杂模型
• 大幅提升性能！
```

### 5.2 LOD设置

```
静态网格体编辑器 → LOD设置

LOD Settings                                           
├─────────────────────────────────────────┤
│ Number of LODs: 3                       │
│                                         │
│ LOD0 (Base): 100% 距离 0     ← 原始模型 │
│ LOD1: 50% 距离 1500          ← 距离1500 │
│ LOD2: 25% 距离 3000          ← 距离3000 │
│                                         │
│ LOD Group: LevelArchitecture            │
└─────────────────────────────────────────┘

LOD Group预设：
• LevelArchitecture  ← 建筑
• Foliage           ← 植被
• Prop              ← 道具
```

### 5.3 自动生成LOD

```
静态网格体编辑器 → LOD菜单 → 生成LOD

【减面设置】
• LOD1: 50% 面数
• LOD2: 25% 面数
• LOD3: 12% 面数

【注意事项】
• 自动生成可能有瑕疵
• 重要模型建议手动制作LOD
• 在视口中测试切换效果
```

## 六、Nanite虚拟几何体

### 6.1 Nanite简介

```
【传统方式的问题】
• 模型面数多了卡顿
• 需要手动做LOD
• 远处模糊，近处锯齿

【Nanite解决方案】
• 直接导入高面数模型（数百万面）
• 自动根据距离调整细节
• 没有LOD切换的"跳变"
• 性能几乎不受面数影响

┌─────────────────────────────────────────────┐
│  传统LOD            vs          Nanite      │
├─────────────────────────────────────────────┤
│                                             │
│   LOD0  ─→  LOD1 ─→ LOD2       无缝过渡     │
│   (跳变明显)                    (平滑渐变)  │
│                                             │
│   需要手动制作LOD              自动处理     │
│   多份模型文件                 单个资源     │
│                                             │
└─────────────────────────────────────────────┘
```

### 6.2 Nanite启用方法

```
导入时启用：
FBX导入选项 → 几何体 → ☑ 启用Nanite

导入后启用：
静态网格体编辑器 → 细节面板 → Nanite设置 → ☑ 启用Nanite支持

注意事项：
• 透明材质不支持Nanite
• 变形/动画不支持Nanite
• 碰撞需要单独设置
• 适合：建筑、石头、静态道具
• 不适合：玻璃、树叶、会动的物体
```

### 6.3 Nanite限制与解决

| 限制 | 说明 | 解决方案 |
|------|------|----------|
| 透明材质 | 不支持 | 关闭Nanite |
| 变形动画 | 不支持 | 用骨骼网格体 |
| 需要碰撞 | 不自动生成 | 手动设置碰撞 |
| 小物体 | 可能负收益 | 面数<2000不开 |

## 七、模型优化检查清单

### 7.1 常见面数参考

| 类型 | 推荐面数 | LOD1 | LOD2 |
|------|----------|------|------|
| 小道具（杯子） | 100-500 | 50 | 25 |
| 中等道具（椅子） | 500-2000 | 500 | 100 |
| 大道具（车辆） | 2000-10000 | 2000 | 500 |
| 建筑（房屋） | 5000-20000 | 5000 | 1000 |
| 地形细节 | Nanite | 自动 | 自动 |

### 7.2 优化检查清单

```
✅ 导入前检查：
  □ 删除无用的顶点/面
  □ 确保法线朝外
  □ UV展开正确（无重叠）
  □ 原点在合适位置（通常底部中心）
  □ 应用所有变换

✅ 导入后检查：
  □ 大小合适
  □ 材质正确
  □ 碰撞生成
  □ LOD设置（如需要）
  □ Nanite启用（适合的话）

💡 记住：能看清就行，不要过度建模！
```
