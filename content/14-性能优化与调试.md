## 一、性能分析工具

### 1.1 统计数据面板

```
【常用统计命令】

按 ~ 键打开控制台，输入：

stat fps         ← 显示帧率
stat unit        ← 显示各线程耗时
stat gpu         ← GPU详细开销
stat memory      ← 内存使用情况
stat rhi         ← RHI统计
stat scenerendering ← 渲染统计
```

### 1.2 性能分析器

```
窗口 → 开发者工具 → 会话前端

功能：
• CPU分析
• GPU分析
• 内存分析
• 渲染线程分析
```

## 二、常见性能瓶颈

### 2.1 CPU瓶颈

```
【常见原因】
• 蓝图Tick太多
• 物理计算过多
• AI逻辑复杂
• 绘制调用（Draw Call）太多

【解决方案】
• 减少Tick频率
• 合并静态网格体
• 使用实例化静态网格
• LOD优化
```

### 2.2 GPU瓶颈

```
【常见原因】
• 面数过高
• 材质太复杂
• 后处理开销大
• 半透明物体太多

【解决方案】
• 使用Nanite
• 简化材质
• 降低后处理质量
• 减少半透明物体
```

### 2.3 内存瓶颈

```
【常见原因】
• 贴图分辨率太高
• 资源没有流送
• 重复加载资源

【解决方案】
• 降低贴图分辨率
• 启用纹理流送
• 资源去重
```

## 三、场景优化检查清单

### 3.1 模型优化

```
□ 使用Nanite（适合的话）
□ 设置合理的LOD
□ 合并小物件
□ 移除不可见面
□ 优化碰撞复杂度
```

### 3.2 材质优化

```
□ 使用材质实例而非多个母材质
□ 减少半透明材质数量
□ 贴图压缩设置正确
□ 避免复杂材质计算
□ Two Sided只在需要时开启
```

### 3.3 光照优化

```
□ 静态光源烘焙而非实时
□ 控制光源数量（重叠不超过3-4个）
□ 合理设置光照范围
□ 使用Lumen时注意距离场优化
□ 阴影质量适当降低
```

### 3.4 植被优化

```
□ 使用Grass Type而非手动刷草
□ 植被LOD设置合理
□ 控制植被密度
□ 远距离剔除
```

## 四、打包与发布

### 4.1 打包设置

```
文件 → 打包项目 → 选择平台

【Windows打包设置】
• 开发/发布模式
• 包含编辑器内容：否
• 压缩内容：是
• 共享材质着色器代码：是
```

### 4.2 发布前检查

```
□ 所有关卡已保存
□ 光照已构建（使用产品级质量）
□ 无错误/警告（尽可能）
□ 性能测试通过
□ 所有资源已迁移到项目内
□ 启动关卡设置正确
```
